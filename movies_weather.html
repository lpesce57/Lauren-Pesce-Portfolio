Movie & Weather API Data Analysis

Description: A Python data-analysis project exploring relationships between film-release success and Ann Arbor weather conditions using multiple APIs.

Class: SI 206: Data-Oriented Programming

Tools Utilized: Python, SQL, JSON, Matplotlib, Rotten Tomatoes API, OMDB API, IMDB API, OpenWeather API

Project Duration: November – December 2023

My Role: Team member in a group of three, independently handled API data cleaning, merging, and correlation analysis

View full code on GitHub: Repository


Problem Statement

This project explored whether weather conditions influence movie popularity by comparing Ann Arbor’s weather on each release day to box-office performance and user ratings. API data provided objective, large-scale evidence to identify any patterns.


About the Course

SI 206 introduces Python programming and data-driven development. Students learn to consume APIs, design databases, and visualize results to solve real-world problems with clean, modular code.


Data Sources & Analysis

I pulled one API for historical weather and three for movie metadata, broadening coverage and reducing single-source bias. Scripts cleaned, merged, and stored the information in SQLite, then SQL joins aligned each film with its local weather. Finally, Matplotlib visualizations and basic statistics revealed potential correlations.


Correlation Between Duration and Rating

Key Findings

  • Weather showed minimal correlation with rating or revenue, though colder days loosely aligned with higher-rated titles.
  • Movie duration had a weak positive correlation with rating, with several notable outliers.
  • Substantial preprocessing was required due to inconsistent API schemas.

Final Result

The project demonstrates multi-API integration, efficient data storage, and clear statistical storytelling. Read the full README for implementation details.
