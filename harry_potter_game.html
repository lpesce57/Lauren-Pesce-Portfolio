Harry Potter Games

Project Detail

Description: A magical two-part site featuring a 20-minute character-guessing challenge and a spell duel against Professor Snape.

Class: SI 379: Building Interactive Applications

Tools Utilized: HTML, CSS, JavaScript, HP-API, Groq API

Project Duration: April – May 2025

Explore Site: Live Demo

Problem Statement

For our final project, we were given full creative freedom to build a website that demonstrated new technical skills alongside concepts we’d learned throughout the semester. As a lifelong Harry Potter fan, I set out to transform Wizarding-world trivia into engaging, browser-based games powered by external APIs and creative interactivity.

About the Course

SI 379 focuses on building interactive web applications with HTML, CSS, JavaScript, and React. Coursework covers topics such as LocalStorage, package management, and real-world API integration to practice modern development workflows.

Analysis

I explored several Harry Potter APIs before selecting HP-API (character & spell data) and Groq API (AI spell adjudication) for their simplicity and impact. Extensive console debugging and user-testing informed pacing, timer logic, and input handling.

Analysis Methods

Prototype testing, console logging, and iterative UI reviews guided refinement of the character-guessing grid and duel mechanics. JSON parsing and responsive-design checks ensured smooth gameplay across devices.

Design

The visual theme draws from the Marauder’s Map. The landing page presents both games with concise descriptions and entry points. Animations, confetti bursts, health bars, and hover states amplify the playful tone.

Design Evolution

The duel originally lacked a health bar and AI logic. Iterations introduced the Groq API for spell resolution and a responsive health bar for clarity. In the guessing game, nickname recognition and pause/resume functionality were added to improve usability and engagement.

Key Design Decisions

  • Groq API decides duel outcomes based on spell comparisons.
  • HP-API supplies character data and spell lists.
  • Custom timer logic with pause/resume for the guessing challenge.
  • Visual feedback (confetti, character images) celebrates milestones.
  • Accessible inputs, error messaging, and responsive layout.

Final Result

Two fully functional browser games: a timed character-guessing challenge and an AI-driven spell duel against Snape — both built from scratch and optimized for desktop and mobile.
