Piazza Machine-Learning Post Classifier

Description: A C++ text-classification tool that automatically labels Piazza discussion-board posts by topic.

Class: EECS 280: Programming & Data Structures

Tools Utilized: C++, STL, Xcode / clang++

Project Duration: Nov – Dec 2023

My Role: Solo developer & algorithm designer (built on minimal starter code)

Project Requirements: View specifications

Problem Statement

Piazza threads for large courses quickly become cluttered. Students struggle to locate answers and instructors lose time responding to duplicates. A simple automatic classifier can streamline this process, saving time for both students and instructors.

About the Course

EECS 280 is Michigan’s foundational C++ course. After an introductory CS course, students tackle larger projects that combine data structures, algorithms, and clean software engineering in real-world scenarios.

Implementation Overview

I re-implemented Binary Search Tree and Map classes from scratch to learn their internals, then swapped to the STL unordered_map for performance in the final classifier. The core class trains on a CSV using a bag-of-words model: word order is ignored, but token frequencies per label are tracked. It outputs log-prior and log-likelihood values in debug mode and predicts labels for unseen posts with a log-probability score.

Running from the Command Line

The program accepts three arguments: a training file, a testing file, and an optional --debug flag. The training CSV lists each post’s label followed by its text; the test file contains unlabeled posts. With --debug enabled, the classifier prints additional statistics after training.

Results & Availability

The classifier achieved ≈ 90 % accuracy on the blind dataset and a 100 % score on the course autograder. Course policy forbids publishing the full source, but I’m happy to schedule a short live demo—just reach out!
